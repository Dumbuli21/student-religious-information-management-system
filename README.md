<h1>Student Religious Information Management System (SRIMS)</h1>

<h2>📌 About SRIMS</h2>
<p>The <strong>Student Religious Information Management System (SRIMS)</strong> is a web-based application designed to manage religious-related information for students within a university or college environment.</p>
<p>The system provides a centralized platform where different religious groups can operate independently within a single system. Each religion can manage its own students, announcements, and events without interfering with other religions.</p>
<p>The system promotes <strong>inclusivity, organization, and effective communication</strong> among students and religious organizations within the institution.</p>

<h2>👥 User Roles</h2>
<table>
    <tr>
        <th>Role</th>
        <th>Description</th>
    </tr>
    <tr>
        <td><strong>Super Admin</strong></td>
        <td>Has full access to the system. Manages religions and system settings. Requires basic computer literacy.</td>
    </tr>
    <tr>
        <td><strong>Religious Admin</strong></td>
        <td>Manages information for a specific religion. Can create and manage Sub-Administrators within the same religion. Oversees student registration, announcements, and events. Views reports and attendance.</td>
    </tr>
    <tr>
        <td><strong>Sub Admin</strong></td>
        <td>Assigned by the Religious Administrator. Belongs to a specific religion only. Assists in registering and managing students. Manages news, announcements, and events for the assigned religion. Cannot access data of other religions.</td>
    </tr>
    <tr>
        <td><strong>Student (User)</strong></td>
        <td>Views information related to their religion. Registers for events. Submits feedback.</td>
    </tr>
</table>
